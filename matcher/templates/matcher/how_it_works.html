How It Works

A beginner-friendly explanation of vector databases and semantic search.

The Problem with Regular Search

Traditional search engines look for exact keywords. If you type "ICU nurse" but a job listing says "critical care healthcare professional", the search misses it - even though they mean the same thing.

❌ Keyword Search:
Input: "ICU nurse experience"
Misses: "Critical Care Healthcare Professional" (different words, same meaning)
🧮

What is a Vector Embedding?

A vector embedding is a list of numbers that represents the meaning of a piece of text. Words or sentences with similar meanings get similar numbers - so they end up close to each other in mathematical space.

Example:
"nurse" → [0.12, -0.45, 0.78, ...]
"doctor" → [0.15, -0.41, 0.80, ...]  ← close!
"JavaScript" → [0.92, 0.11, -0.33, ...]  ← far away

We use a model called all-MiniLM-L6-v2 (from sentence-transformers) to generate these vectors. It's a lightweight but powerful AI model that's free to use.

🗄️

What is ChromaDB?

ChromaDB is a vector database - a special type of database designed to store and search embedding vectors efficiently. Unlike regular databases that compare text, ChromaDB compares numbers to find the most mathematically similar entries.

  • ✅ Free and open-source
  • ✅ Runs completely on your local machine (no cloud needed)
  • ✅ Very fast - searches thousands of vectors in milliseconds
  • ✅ Easy to install: just pip install chromadb
🔄

What is Cosine Similarity?

Cosine similarity measures the angle between two vectors. Two vectors pointing in the same direction have a similarity of 1.0 (100%), meaning they're identical in meaning. Vectors pointing in opposite directions have a similarity of 0.0 (0%).

Formula: similarity = (A · B) / (|A| × |B|)

Result: A number between 0 and 1
→ 0.9 = 90% similar (excellent match)
→ 0.5 = 50% similar (moderate match)
→ 0.1 = 10% similar (poor match)
